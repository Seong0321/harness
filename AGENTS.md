# AGENTS.md

이 파일은 AI 에이전트가 이 프로젝트에서 작업을 수행할 때 준수해야 하는 행동 지침과 프로젝트 정보를 담고 있습니다.

## 1. 프로젝트 개요 (Project Context)
- **프로젝트 명**: Hideout
- **목적**: 프론트엔드 및 웹 개발 학습 및 실습을 위한 개인 저장소입니다.
- **기술 스택**: HTML5, Vanilla CSS, JavaScript

## 2. 답변 및 소통 스타일 (Communication & Response Style)
에이전트는 사용자와 소통하고 결과물을 전달할 때 다음 규칙을 엄격히 준수해야 합니다:

- **정중하고 친절한 태도 유지**: 모든 상호작용에서 정중하고 배려 깊은 어조(톤앤매너)를 유지합니다.
- **상세한 설명과 충분한 분량 (최소 100자 이상)**: 
  - 사용자의 요청에 응답하거나 코드 변경 사항을 설명할 때, 단순히 코드만 던지거나 단답형으로 대답하지 마십시오.
  - 해당 변경을 수행한 이유, 코드의 작동 원리, 적용 방법, 혹은 필요한 추가 작업 등을 논리적이고 친절하게 작성해야 하며, 설명 부분은 **최소 100자 이상**의 완성도 높은 문장으로 기술해야 합니다.
- **과정의 투명성**: 문제 해결 과정에서 의사결정의 배경이나 대안을 명시하여 사용자가 에이전트의 진행 상황을 완벽히 이해할 수 있도록 돕습니다.

## 3. 코딩 표준 및 규칙 (Coding Standards & Conventions)
- **시맨틱 HTML 사용**: 웹 접근성과 SEO를 위해 적절한 HTML5 시맨틱 태그(`header`, `nav`, `main`, `section`, `article`, `footer` 등)를 적극 사용합니다.
- **순수 CSS(Vanilla CSS) 우선**: 외부 스타일 라이브러리 사용을 지양하고, 일관성 있는 디자인 시스템(CSS 변수 활용 등)을 직접 구축하여 사용합니다.
- **주석 및 문서화**: 모든 수정 사항에는 원래의 주석과 문서 구조를 최대한 보존하고, 필요 시 가독성 높은 주석을 추가합니다.

## 4. 행동 범위 및 제약 사항 (Boundaries & Constraints)
- **⚠️ 확인 필요 (Ask First)**:
  - 중요 디렉토리 구조의 변경
  - 외부 라이브러리나 패키지의 대대적인 추가/삭제
- **🚫 절대 금지 (Never)**:
  - 사용자 승인 없이 로컬 환경의 자격 증명(API Key 등)이나 개인정보를 노출/커밋하는 행위
  - 불필요하게 복잡하거나 오버엔지니어링된 코드를 도입하는 행위

## 5. 주요 명령어 (Common Commands)
- 이 저장소는 정적 웹 페이지 위주이므로 필요한 경우 `Live Server` 등 로컬 서버 실행 도구를 추천하거나 안내합니다.
